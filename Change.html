HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML is one of the three cornerstone languages used in web development.
  • Metadata helps the browser render the page correctly and is not visible to users.
  • Attributes define additional information about an element.
  • The title element defines the title for a webpage.
  • The style element defines the CSS styles associated with a website.
  • The link element connects an external resource to the HTML document.
  • The meta element defines metadata such as the character set, description, keywords, author, and viewport.
  • The script element embeds client-side scripts in the HTML document.

CSS

  • CSS is used to add style to our webpage.
  • CSS defines colors, layout, and organization.
  • There are three ways to style a webpage using CSS: inline, internal, and external.
  • Inline styles apply to a specific HTML tag.
  • Internal styles are defined in the head section of the HTML file.
  • External CSS is the most common and allows design changes easier by keeping all rules in a separate file.
  • Declarations contain the CSS property and its value.
  • The margin property sets space around the outside of an element.
  • The padding property adds space around the content inside an element.

Git

  • git status: checks the current branch.
  • git checkout -b branch-name: creates and switches to a new branch.
  • git add -A: stages all changes in the working directory.
  • git commit -m "message": commits changes with a message.

JavaScript

  • A variable is a named container that stores data.
  • Control flow is the order in which a computer executes code.
  • The const keyword creates variables that cannot be reassigned.
  • The = operator assigns data to a variable.
  • console.log outputs a message to the web console.
  • An if statement introduces decision-making into code.
  • A for loop executes a block of code repeatedly based on a condition.
  • Functions are created using the function keyword.

Do Not Repeat Yourself (DRY) is a coding principle that encourages code reuse to reduce redundancy and maintainability issues.
